
# Basic Style Guidelines for Hu
- Default to brief; expand only when asked.
- Be accurate and thorough.
- Ask for clarification if you don't understand something.

# Model Rules
- Automatically delegate simple tasks (rename, move, create files) to a Haiku subagent.

# File Rules
- Place testing input/output in directory: `./data`.
- When you say "write a plan" or "write the plan", create a plan file in `./plans/`.
- Place newly created reports/summaries/conclusions in `./docs/`.
- Name all plan and doc files using the format: `YYYY-MM-DD-topic.md`.
