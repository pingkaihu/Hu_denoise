# 核心技術文件疑義與盲點分析 (Critique)

這份文件基於影像處理演算法與深度學習的專業視角，針對專案中關於 Noise2Void、Deep Image Prior, Anscombe 轉換以及相關的混合噪聲 (Speckle + Shot Noise) 策略，提出 5 個關鍵的疑義與潛在挑戰。這些問題在應對高階或低劑量 SEM 真實應用時，可能會成為效能的瓶頸。

## 1. 理論冗餘：GAT 轉換與 PN2V 的疊床架屋？

在針對混合噪聲的建議中，文件最強烈推薦的終極方案是 **GAT (廣義 Anscombe 轉換) + PN2V**。

*   **核心疑義**：GAT 的數學目的，是要把 Poisson-Gaussian 的「訊號相依性」完全抹平，轉化為方差為 1 的**純加性高斯白噪聲 (AWGN)**。
*   如果 GAT 轉換真的有完美發揮作用，此時的殘餘噪聲理應是高斯分佈，那麼**其實使用原版 N2V (甚至更簡單的 L2 Loss) 就已經是數學最優解了**，不需動用到 PN2V。反過來說，PN2V 最強悍的地方在於它能學習「複雜、非高斯、訊號相依」的 Noise Model。先做 GAT 再接 PN2V，在數學上有點多此一舉；甚至 PN2V 的高斯混合模型(GMM)去擬合 GAT 轉換後本已接近高斯的訊號時，可能會過度擬合 GAT 在低訊號區的轉換誤差，將其誤認為特徵去學習。

## 2. 生死攸關的極低劑量區：Log 與 Anscombe 在「低數值區」的極度不穩定

文件針對 Speckle 建議了 `Log` 轉換，或針對 Shot Noise 的 `GAT` 轉換。

*   **核心疑義**：文獻指出，在極低電子劑量（Extreme Low-dose SEM）下，像素計數 (Photon/Electron Counts) 經常會在 0 到 3 之間跳動。
*   此時，這類非線性轉換會嚴重失效。文件提議用 `np.log1p` 或 `+ 1e-8` 來避免數學錯誤，但在計數接近 0 的暗區，稍微一點噪聲擾動，過 Log 或平方根後，其誤差的放大將是非線性的，從而產生強烈的偏差(Bias)。這導致逆轉換 (Inverse Transform) 算出來的黑背景嚴重失真發灰，或充滿假影。目前的策略中，並未看到對「極低計數邊界條件」有足夠強健的保護或截斷演算法。

## 3. Self2Self 處理空間相關 (Spatially Correlated) 噪聲的過度樂觀

文件中提到，若不知噪聲模型，可用 `Self2Self` 作為備案，並聲稱它可以解決 Speckle 問題。

*   **核心疑義**：S2S 依賴 Dropout (隨機丟棄部分像素) 來建造 Ensemble。但這個方法隱含了一個極強的物理先驗：「**噪聲的空間變異（Grain Size）不得大於像素獨立性**」。
*   如果 SEM 倍率極高，導致 Speckle 斑塊 (Grain) 橫跨了 3×3 甚至 5×5 的像素（高度空間相關），那麼即便 Dropout 掉 30% (`0.3`) 的像素，網路依然可以輕易從周圍「未被 Drop 掉」的像素中讀取到一致的 Speckle 殘影。這會導致 S2S 在強烈空間相關的場景下失效，**錯把 Speckle 的連片大斑塊當作「真實訊號紋理」給保留下來**。

## 4. 擴大 blind-spot 對「奈米級解析度」的毀滅性打擊

針對空間相關性強的 Speckle 噪聲，文件中建議：「將 blind-spot 擴大至 3×3 或 5×5」。

*   **核心疑義**：Blind-spot 的神經網路核心邏輯是「用周圍圈圍的像素，來預測中心像素的值」。
*   如果硬性地把盲區擴大到 5×5，等於強迫演算法從 2-3 個像素外圍的距離去猜測中心點的亮暗。對於 SEM 這種專門用來觀測微觀結構（可能一條奈米碳管的寬度就只有 1~2 個像素）的精密儀器而言，擴大盲區必然導致**高頻細節 (微小邊緣與極細結構) 被嚴重抹平 (Oversmoothing)**。文件對於這層「消除噪聲空間相關性」與「維持訊號空間極限解析度」的不可逆 Trade-off 描述得太過輕描淡寫。

## 5. 對於 Batch Prediction (多圖批次轉移) 穩定性的存疑

在運算資源比較中，文件強調 N2V / PN2V 訓練後具有極高的「可重用性 (Batching)」，聲稱練好第一張可以高速套用給後續的 100 張影像。

*   **核心疑義**：這個策略在自然光學攝影上可能成立，但在晶圓檢測或材料分析的 SEM 實際產線上風險極高。由於**樣品表面帶電 (Charging effect)**、長期掃描造成的探針電流漂移 (Beam current drift)、或散焦，同一批次的前後照片，其 Base Noise Level (尤其是讀出噪聲 $\sigma\_read$ 與背景亮度) 經常是動態浮動的。
*   如果用第一張圖練出的 PN2V Noise Model 或 N2V 先驗，強硬套入第 50 張稍微過曝或變暗的圖裡，非常容易發生嚴重的去噪偏差或是幻覺 (Hallucination)。若真要做到工業級的批量連續處理，系統可能必須引入強健的**全域直方圖匹配 (Global Histogram Matching)** 或**動態 $\sigma\_read$ 即時校正機制**。
